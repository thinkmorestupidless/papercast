# Implementation Plan: Research Podcast Creator

**Branch**: `001-research-podcast-creator` | **Date**: 2026-03-17 | **Spec**: [spec.md](spec.md)

## Summary

Users submit a search query; the system searches arXiv, PubMed, and Semantic Scholar, summarises the returned papers via AI agents, generates a podcast script from those summaries, and streams an audio recording from ElevenLabs. A `PodcastCreationWorkflow` orchestrates the three-step pipeline (search → summarise → script); audio generation is handled on demand by the HTTP endpoint.

---

**Language/Version**: Java 21

**Storage**: Workflow state (in-memory with Akka persistence); no external database required
**Testing**: JUnit 5, Akka TestKit (`TestKitSupport`), `EventSourcedTestKit`, `TestModelProvider`
**Target Platform**: Akka cloud service (local dev via `mvn exec:java`)
**Project Type**: Web service
**Performance Goals**: Paper results within 60s; audio delivered within 120s of request
**Constraints**: ElevenLabs API key required; NCBI and Semantic Scholar keys recommended; no binary audio stored in workflow state
**Scale/Scope**: Single-user sessions; each session is an independent workflow instance

## Component Design

### `PodcastCreationWorkflow`

**Type**: `Workflow<PodcastCreationState>`
**Component ID**: `podcast-creation`

**Command handlers**:

| Method | Parameter | Returns | Description |
|---|---|---|---|
| `create(CreateCommand)` | `record CreateCommand(String query)` | `Effect<Done>` | Starts pipeline; transitions to `searchPapersStep` |
| `getStatus()` | none | `Effect<PodcastCreationState>` | Returns current state for polling |

**Steps**:

| Step method | Annotation | Action | Timeout | Recovery |
|---|---|---|---|---|
| `searchPapersStep` | `@StepName("search-papers")` | HTTP calls to Semantic Scholar, arXiv, PubMed; deduplicate; top 10 | 30s | `maxRetries(2)` → `failedStep` |
| `summarisePapersStep` | `@StepName("summarise-papers")` | Call `PaperSummaryAgent` with all papers | 60s | `maxRetries(1)` → `failedStep` |
| `generateScriptStep` | `@StepName("generate-script")` | Call `PodcastScriptAgent` with summaries | 60s | `maxRetries(1)` → `failedStep` |
| `failedStep` | `@StepName("failed")` | Updates state with `FAILED` status | 5s | — |

**Session ID**: `commandContext().workflowId()` — shared between both agents for context continuity.

---

### `PaperSummaryAgent`

**Type**: `Agent`
**Component ID**: `paper-summary`

**Command handler**: `summarise(SummariseRequest) → Effect<SummariseResponse>`

- System prompt: instructs the model to produce concise 2–4 sentence plain-language summaries for a non-specialist audience, one per paper in input order.
- Input: `record SummariseRequest(List<Paper> papers)`
- Output: `record SummariseResponse(List<PaperSummary> summaries)` — uses `responseConformsTo(SummariseResponse.class)`.
- Error handling: `.onFailure(t -> new SummariseResponse(List.of()))` — returns empty list on failure; workflow detects and transitions to `failedStep`.

---

### `PodcastScriptAgent`

**Type**: `Agent`
**Component ID**: `podcast-script`

**Command handler**: `generateScript(ScriptRequest) → Effect<PodcastScript>`

- System prompt: instructs the model to write an engaging, conversational podcast script — clear introduction, one narrative segment per paper, and a closing reflection on the research landscape.
- Input: `record ScriptRequest(String topic, List<PaperSummary> summaries)`
- Output: `PodcastScript` — uses `responseConformsTo(PodcastScript.class)`.
- Error handling: `.onFailure(t -> ...)` — returns a minimal fallback script; workflow detects empty segments and transitions to `failedStep`.

---

### `ResearchPodcastEndpoint`

**Type**: HTTP Endpoint
**Base path**: `/podcast`
**ACL**: `@Acl(allow = @Acl.Matcher(service = "*"))` (internal only by default)

**Methods**:

| HTTP | Path | Handler | Description |
|---|---|---|---|
| POST | `/podcast` | `createPodcast(CreatePodcastRequest)` | Starts workflow; returns 201 with `workflowId` |
| GET | `/podcast/{id}/status` | `getStatus(String id)` | Polls workflow state; returns status DTO |
| GET | `/podcast/{id}/script` | `getScript(String id)` | Returns script when complete; 409 if not ready |
| GET | `/podcast/{id}/audio` | `getAudio(String id)` | Fetches script, calls ElevenLabs, streams MP3 |

**ElevenLabs integration** (within `getAudio`):
1. Fetch `PodcastCreationState` via `componentClient`; return 409 if status ≠ `COMPLETE`.
2. Build full script text from `PodcastScript` fields.
3. Call `POST https://api.elevenlabs.io/v1/text-to-speech/{voiceId}/stream` with `xi-api-key` header.
4. Stream response bytes to client as `HttpResponse` with `Content-Type: audio/mpeg`.
5. On ElevenLabs failure: return 502 with JSON body pointing to `/script` endpoint.

## Configuration

```hocon
# src/main/resources/application.conf
elevenlabs {
  api-key  = ${ELEVENLABS_API_KEY}
  voice-id = "21m00Tcm4TlvDq8ikWAM"   # Rachel
  model-id = "eleven_turbo_v2_5"
}

paper-search {
  ncbi-api-key = ${?NCBI_API_KEY}      # optional; increases rate limit
  ncbi-tool    = "research-podcast-creator"
  ncbi-email   = "contact@example.com"
  s2-api-key   = ${?S2_API_KEY}        # optional; improves rate limit
  max-results  = 10
}
```

`ELEVENLABS_API_KEY` is required at startup. `NCBI_API_KEY` and `S2_API_KEY` are optional but strongly recommended for reliable rate limits in production.
