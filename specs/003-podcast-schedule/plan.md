# Implementation Plan: Scheduled Podcast Creation

**Branch**: `003-podcast-schedule` | **Date**: 2026-03-18 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/003-podcast-schedule/spec.md`

## Summary

Allow authenticated users to configure recurring podcast schedules (DAILY / WEEKLY / MONTHLY with UTC times). At each scheduled time, a `TimedAction` fires, triggers the existing `PodcastCreationWorkflow` (feature 001), records the run, and reschedules the next timer. Schedule configuration is stored in a `KeyValueEntity`. A `View` enables per-user schedule listing. The endpoint enforces a 10-schedule-per-user limit and manages timer lifecycle (create, cancel, reschedule).

## Technical Context

**Language/Version**: Java 21
**Primary Dependencies**: Akka SDK 3.5.16 (no new external dependencies required)
**Storage**: Akka Key Value Entity (schedule state + bounded execution history)
**Testing**: JUnit 5, Akka TestKit, `KeyValueEntityTestKit` for unit tests, `TestKitSupport` for integration tests
**Target Platform**: JVM / Akka platform (local dev + cloud deploy)
**Project Type**: Akka web service (extending existing service)
**Performance Goals**: Schedule fires within 5 minutes of configured UTC time (SC-002); `TimerScheduler` precision is sub-minute
**Constraints**: All schedule times UTC; `dayOfMonth` capped at 28; max 10 schedules per user
**Scale/Scope**: 1,000 concurrent scheduled users (SC-006); one outstanding timer per schedule

## Component Design

### PodcastSchedule (domain record)

```java
// src/main/java/com/example/domain/PodcastSchedule.java
public record PodcastSchedule(
    String scheduleId,
    String userId,
    String searchTerms,
    Cadence cadence,
    int timeHour,
    int timeMinute,
    DayOfWeek dayOfWeek,    // non-null for WEEKLY; null otherwise
    Integer dayOfMonth,     // non-null for MONTHLY (1–28); null otherwise
    Status status,
    Instant createdAt,
    Instant lastModifiedAt,
    Instant nextRunAt,      // null when PAUSED
    List<ScheduleRun> recentRuns   // newest first, max 100
) {
    public enum Cadence { DAILY, WEEKLY, MONTHLY }
    public enum Status  { ACTIVE, PAUSED }

    public record ScheduleRun(
        String runId,
        Instant triggeredAt,
        String workflowId,
        String workflowStatusUrl
    ) {}

    // Factory
    public static PodcastSchedule create(String scheduleId, String userId, String searchTerms,
        Cadence cadence, int timeHour, int timeMinute,
        DayOfWeek dayOfWeek, Integer dayOfMonth, Instant now) { ... }

    // Pure domain method — no Akka dependencies
    public Instant nextRunAfter(Instant from) { ... }
        // DAILY:   next HH:MM UTC after `from`
        // WEEKLY:  next dayOfWeek + HH:MM UTC after `from`
        // MONTHLY: next dayOfMonth + HH:MM UTC after `from`;
        //          skip month if day > month length

    // Mutation helpers
    public PodcastSchedule withPaused() { ... }
    public PodcastSchedule withResumed(Instant nextRunAt) { ... }
    public PodcastSchedule withUpdated(String searchTerms, Cadence cadence,
        int timeHour, int timeMinute, DayOfWeek dayOfWeek, Integer dayOfMonth,
        Instant nextRunAt, Instant now) { ... }
    public PodcastSchedule withRunRecorded(ScheduleRun run, Instant nextRunAt) { ... }
        // prepend run; trim list to 100 entries; set nextRunAt
}
```

### PodcastScheduleTrigger (Timed Action)

```java
// src/main/java/com/example/application/PodcastScheduleTrigger.java
@Component(id = "podcast-schedule-trigger")
public class PodcastScheduleTrigger extends TimedAction {

    // Injected
    private final ComponentClient componentClient;
    private final TimerScheduler timerScheduler;

    public Effect trigger(String scheduleId) {
        // 1. Fetch schedule state
        PodcastSchedule schedule = componentClient.forKeyValueEntity(scheduleId)
                .method(PodcastScheduleEntity::getSchedule).invoke();

        // 2. Short-circuit if null or PAUSED
        if (schedule == null || schedule.status() == PAUSED) {
            return effects().done();
        }

        // 3. Start podcast creation workflow
        String workflowId = UUID.randomUUID().toString();
        componentClient.forWorkflow(workflowId)
                .method(PodcastCreationWorkflow::create)
                .invoke(new CreateCommand(schedule.searchTerms()));

        // 4. Record run and advance nextRunAt
        Instant now = Instant.now();
        Instant nextRunAt = schedule.nextRunAfter(now);
        componentClient.forKeyValueEntity(scheduleId)
                .method(PodcastScheduleEntity::recordRun)
                .invoke(new RecordRun(UUID.randomUUID().toString(), now,
                        workflowId, "/podcast/" + workflowId + "/status", nextRunAt));

        // 5. Schedule next timer
        Duration delay = Duration.between(now, nextRunAt);
        timerScheduler.createSingleTimer(
                "podcast-schedule-" + scheduleId, delay,
                componentClient.forTimedAction()
                        .method(PodcastScheduleTrigger::trigger)
                        .deferred(scheduleId));

        return effects().done();
    }
}
```

### SchedulesByOwnerView (View)

```java
// src/main/java/com/example/application/SchedulesByOwnerView.java
public class SchedulesByOwnerView extends View {

    public record ScheduleSummary(String scheduleId, String userId, String searchTerms,
        String cadence, String status, Instant nextRunAt, Instant createdAt) {}

    public record ScheduleSummaries(List<ScheduleSummary> schedules) {}

    @Consume.FromKeyValueEntity(PodcastScheduleEntity.class)
    public static class ScheduleRows extends TableUpdater<ScheduleSummary> {
        public Effect<ScheduleSummary> onUpdate(PodcastSchedule state) {
            if (state == null) return effects().deleteRow();
            return effects().updateRow(new ScheduleSummary(
                state.scheduleId(), state.userId(), state.searchTerms(),
                state.cadence().name(), state.status().name(),
                state.nextRunAt(), state.createdAt()));
        }
    }

    @Query("SELECT * AS schedules FROM schedule_rows WHERE userId = :userId ORDER BY createdAt DESC")
    public QueryEffect<ScheduleSummaries> getByOwner(String userId) {
        return queryResult();
    }
}
```

### PodcastScheduleEndpoint (HTTP Endpoint)

```java
// src/main/java/com/example/api/PodcastScheduleEndpoint.java
@HttpEndpoint("/schedules")
@Acl(allow = @Acl.Matcher(principal = Acl.Principal.INTERNET))
@JWT(validate = JWT.JwtMethodMode.BEARER_TOKEN)
public class PodcastScheduleEndpoint extends AbstractHttpEndpoint {

    private final ComponentClient componentClient;
    private final TimerScheduler timerScheduler;

    // Request/Response records
    public record CreateScheduleRequest(String searchTerms, String cadence,
        int timeHour, int timeMinute, String dayOfWeek, Integer dayOfMonth) {}

    public record UpdateScheduleRequest(String searchTerms, String cadence,
        int timeHour, int timeMinute, String dayOfWeek, Integer dayOfMonth) {}

    public record RunsResponse(List<PodcastSchedule.ScheduleRun> runs) {}

    // Routes
    @Post    POST /schedules                    createSchedule(CreateScheduleRequest) → 201 ScheduleCreatedResponse | 400
    @Get     GET  /schedules                    listSchedules() → 200 ScheduleSummaries
    @Get     GET  /schedules/{id}               getSchedule(String id) → 200 PodcastSchedule | 404
    @Patch   PATCH /schedules/{id}/pause        pauseSchedule(String id) → 200 | 404
    @Patch   PATCH /schedules/{id}/resume       resumeSchedule(String id) → 200 | 404
    @Put     PUT  /schedules/{id}               updateSchedule(String id, UpdateScheduleRequest) → 200 | 404
    @Delete  DELETE /schedules/{id}             deleteSchedule(String id) → 200
    @Get     GET  /schedules/{id}/runs          getScheduleRuns(String id) → 200 RunsResponse | 404
}
```

`createSchedule` flow:
1. Extract `userId` from JWT `sub` claim.
2. Query `SchedulesByOwnerView` — reject with 400 if count ≥ 10.
3. Validate request fields (non-blank `searchTerms`, valid cadence, valid time, cadence-appropriate day fields).
4. Generate `scheduleId` (UUID) and compute initial `nextRunAt`.
5. Call `PodcastScheduleEntity::createSchedule`.
6. Create first timer: `timerScheduler.createSingleTimer("podcast-schedule-" + scheduleId, delay, ...)`.
7. Return 201 with `ScheduleCreatedResponse`.

`pauseSchedule` / `deleteSchedule`: call `timerScheduler.cancel("podcast-schedule-" + id)` before entity command.

`resumeSchedule`: recompute `nextRunAt`, call entity, then create new timer.

`updateSchedule`: compute new `nextRunAt`, call entity to update config, cancel old timer, create new timer with updated delay.

Ownership check for all `{id}` routes: fetch entity state and verify `state.userId()` matches authenticated user; return 404 if mismatch (avoids leaking schedule existence).

Timer name helper:
```java
private static String timerName(String scheduleId) {
    return "podcast-schedule-" + scheduleId;
}
```

## Test Strategy

### Unit Tests — `PodcastScheduleTest.java`

Pure domain logic tests (no Akka dependencies):
- `nextRunAfter()` for DAILY — same-day future time; next-day rollover
- `nextRunAfter()` for WEEKLY — same-week future; next-week rollover
- `nextRunAfter()` for MONTHLY — same-month future; next-month rollover; skip short month (day 28 in Feb)
- `withRunRecorded()` — list prepends and trims to 100 entries

### Unit Tests — `PodcastScheduleEntityTest.java`

Uses `KeyValueEntityTestKit<PodcastSchedule, PodcastScheduleEntity>`:
- `createSchedule` on new entity → state is ACTIVE with correct fields
- `createSchedule` on existing entity → error (already exists)
- `pauseSchedule` on ACTIVE → status becomes PAUSED, `nextRunAt` is null
- `pauseSchedule` on PAUSED → error
- `resumeSchedule` on PAUSED → status becomes ACTIVE
- `resumeSchedule` on ACTIVE → error
- `updateSchedule` on existing → fields updated
- `updateSchedule` on non-existent → error
- `deleteSchedule` → state is null
- `recordRun` → run appended, `nextRunAt` advanced
- `recordRun` on null state → no-op (returns Done)
- `getSchedule` on non-existent → error (or empty response; to be decided)

### Integration Tests — `PodcastScheduleEndpointIntegrationTest.java`

Uses `TestKitSupport`, `httpClient`, `bearerTokenWith(...)`:
- `POST /schedules` with valid request → 201 with `scheduleId` and `nextRunAt`
- `POST /schedules` with blank `searchTerms` → 400
- `POST /schedules` with MONTHLY cadence and `dayOfMonth = 29` → 400
- `GET /schedules` returns created schedule in list
- `GET /schedules/{id}` returns schedule (ownership validated)
- `GET /schedules/{id}` for another user's schedule → 404
- `PATCH /schedules/{id}/pause` → 200; subsequent `GET` shows PAUSED
- `PATCH /schedules/{id}/pause` again → 400
- `PATCH /schedules/{id}/resume` → 200; subsequent `GET` shows ACTIVE
- `PUT /schedules/{id}` → 200; GET reflects updated fields
- `DELETE /schedules/{id}` → 200; subsequent `GET` → 404
- `GET /schedules/{id}/runs` → 200 with empty runs list initially
- `POST /schedules` 11 times (10 existing) → 400 (limit reached)
- `DELETE` reduces count, allowing creation again

**Note**: Timer firing behaviour is not tested in integration tests because `TimerScheduler` timers in `TestKitSupport` do not auto-fire. The `PodcastScheduleTrigger` can be invoked directly via `componentClient` in a dedicated scenario (see `quickstart.md` scenario 4).
